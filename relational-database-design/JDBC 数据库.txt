Catalog和Schema是相同的概念
 - Catalog is used with Oracle, DB2 and MySQL
 - Schema is for PostgresSQL

不同的数据库之间存在JDBC Type的型差别，存在Function方法不同
不同数据库对于where in ()判断条件的设置不同

原则上不在数据库中存储非法数据，除非用于测试，程序在DB中存储非法数据前必须提示

TODO. 关于JDBC Connection数据库连接
1. JDBC Statement无法适配不同的数据库的区别，在执行上面需要硬编码
2. JDBC Connection默认是开启AutoCommit
   在自动提交的情况下不能再AutoCommit显式提交: Cannot commit when autoCommit is enabled.
   在取消自动提交的情况下，可以自定义事务提交或者回滚: connection.commit()或connection.rollback();
3. JDBC Connection连接具有Timeout
   一般数据库接收TCP/HTTP的连接具有Timeout超时时间(Connection, Idle, Statement)
   在超时时间内没有完成将会造成异常，导致程序无法继续执行 !

为什么数据库中的表在设计是有外键约束的，但实际建表操作时却可以删除外键
1. 没有外键对于一条需要匹配外键的查询语句而言并不影响
   SELECT * FROM student LEFT JOIN grade ON student.id = grade.id WHERE grade.id = 1;
2. 对于数据库表经常变动update的场景，添加外键约束会影响修改数据
3. 方便添加测试数据到表中，而不用考虑外键对应表的数据
